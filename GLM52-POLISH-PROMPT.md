## 项目概况

这是汐汐酱的个人博客站（F.windEmiko.top），技术栈：
- 后端：FastAPI + SQLAlchemy 2.0 async + MySQL + JWT
- 公网前端：Nuxt 3 (SSR/SSG) + TailwindCSS + Element Plus
- 管理后台：Vite + Vue 3 (独立 SPA, CSR)
- pnpm monorepo，packages/shared 和 packages/ui 供双前端共用
- 右下角 Live2D 角色（Mao 模型，pixi-live2d-display）
- 设计方向：**二次元 + 代码极客风格融合**，玻璃磨砂质感，深色基调

当前项目已在本地开发环境可用：
- 后端：http://localhost:8000
- 公网前端：http://localhost:3000（可能偏移到 3001）
- 管理后台：http://localhost:5173

## 🔴 P0：严重渲染故障（必须修）

### P0-1：博客详情页内容空白
- **文件**：`nuxt-app/pages/blog/[slug].vue`
- **现象**：页面只渲染导航栏和页脚，文章标题/正文/元数据全部不可见
- **可能原因**：asyncData/useAsyncData 数据获取失败、组件条件渲染 v-if 提前短路、`[slug]` 路由参数匹配问题
- **排查方向**：检查 `useAsyncData` 返回的数据结构、API 响应格式是否被 `unwrap` 函数正确解析、组件模板中是否过早 return

### P0-2：资源详情页内容空白
- **文件**：`nuxt-app/pages/resources/[slug].vue`
- **现象**：与博客详情相同——页头页尾在，核心内容不渲染
- **可能原因**：同上，也可能是 `[slug]` 参数包含了中文或特殊字符导致 API 请求失败

### P0-3：登录页完全空白
- **文件**：`nuxt-app/pages/login.vue`
- **现象**：整个页面白屏，无输入框、无按钮、无文案
- **可能原因**：组件未正确导出/导入、ClientOnly 包裹不当导致 SSR 和 CSR 都不渲染、路由配置问题

### P0-4：注册页完全空白
- **文件**：`nuxt-app/pages/register.vue`
- **现象**：与登录页相同
- **可能原因**：同上

### P0-5：404 页面是 Nuxt 默认页面
- **文件**：`nuxt-app/error.vue`（如果不存在则创建）
- **修复**：创建自定义 `error.vue`，继承站点的 AppHeader + AppFooter，中间放自定义 404 图文。保持与全站一致的玻璃磨砂风格。如果是 404 以外的错误码，展示对应状态信息。

## 🟡 P1：功能/体验修复

### P1-1：代码块无语法高亮
- **涉及**：博客详情页的 Markdown 渲染（后端 `service.py` 的 `render_md` 函数 + 前端 MarkdownViewer 组件）
- **要求**：
  - 后端 Markdown 渲染使用 `markdown` 库的 `codehilite` 扩展或换成 `mistune` + 自定义高亮
  - 前端代码块需要深色终端风格背景（如 `#1e1e2e` Catppuccin Mocha 或 `#282a36` Dracula）
  - 行内代码用粉色/橙色高亮背景
  - 支持语言标签（```python、```bash 等）自动识别
  - 代码字体：**JetBrains Mono** 或 **Fira Code**
  - ⚠️ 关键：这是这个博客的灵魂——既然是技术博客，代码块必须是最好看的部分

### P1-2：设计风格注入 — 二次元 + 代码极客融合

当前的视觉风格偏"现代玻璃态科技风"，缺少二次元和极客元素。

#### 二次元方向：

1. **Hero Banner 插画**
   - 首页 Hero 区域需要一个 ACG 风格的背景：可以是星空 + 像素点阵 + 渐变色（紫→蓝→青）
   - 副标题装饰：在"代码 · 游戏 · 创作"文字两侧加小像素图案（如 Minecraft 钻石镐、星形、小狐狸）
   - 如果没有插画素材，用纯 CSS/SVG 的几何图案 + 粒子动画替代

2. **动漫装饰元素**
   - 页面角落或分隔线上添加 subtle 的樱花/星星飘落动画（纯 CSS，不依赖 JS 动画库）
   - 文章卡片 hover 时用微妙的粉色/紫色光晕
   - 标题字体：h1/h2 可以用一种带日语假名感觉的字体声明（如 `'ZCOOL XiaoWei', 'Noto Serif SC'`），正文保持 Sans

3. **站点 Favicon**
   - 简单的小狐狸 logo 或像素风格 `F` 字母图标，用纯 SVG 内联

4. **Live2D 优化**
   - 移动端（<768px）增加收起/展开按钮，默认收起为一个小圆点，点击展开
   - 气泡文本改成萌系台词库（"怎么了喵~"、"别戳啦~"、"嗯哼？"），不要用通用文本

#### 代码极客方向：

1. **代码块终端化**
   - 每个代码块左上角仿 macOS 窗口三色圆点（红黄绿）
   - 代码块标题栏显示语言名称（如 `python`、`bash`）
   - 行号显示（灰色小字）
   - 深色背景 + 亮色语法高亮

2. **全局代码字体装饰**
   - 首页或博客列表页随机展示一行"终端风格"的装饰文本（如 `$ echo "Hello, World"` 或 ASCII art）
   - 页脚加一行 ASCII art 小狐狸

3. **终端绿点缀**
   - 在全局色彩变量中增加一个 `--color-terminal: #00FF41`（Matrix 绿）作为强调色
   - 链接 hover、选中状态、加载动画用这个颜色
   - 但不是全站绿色——只在细节处点缀

4. **"黑客模式"彩蛋**
   - 连续点击页脚 5 次 → 切换到一个暗绿色终端主题（可选，不必高优先级）

### P1-3：玻璃磨砂效果优化

#### 浅色模式增强
- 当前浅色模式下玻璃背景 `rgba(255,255,255,0.65)` 偏白、不够通透
- 改为 `rgba(255,255,255,0.75)` + 增加 `bg-gradient-to-br from-white/15 to-white/5` 渐变叠加
- 浅色模式下的玻璃边框颜色从 `#e5e7eb` 改为偏蓝的 `#dbeafe`

#### 光泽扫过效果
- 为 `.glass-card` 增加 hover 时的 `::after` 伪元素：从左上到右下扫过一道半透明白色光带
- CSS transition 0.4s ease，不要太快

#### 深色模式微调
- 背景从纯黑 `#0B0B1A` 改为带一点点紫的深色 `#0a0a1a`
- 玻璃背景 `rgba(255,255,255,0.04)` → `rgba(255,255,255,0.05)`（太暗了加点光）

### P1-4：移动端优化

- Live2D 角色增加收起/展开按钮（<768px 时显示）
- ChatPanel 移动端高度 `h-[520px]` → `h-[380px]`，宽度保持 `w-[90vw]`
- 资源卡片标题移动端允许换行（`line-clamp-2` 替代 `truncate`）
- 下载站移动端文件夹树默认折叠，点击"浏览文件夹"按钮展开
- 确保所有可点击元素最小触控面积 44×44px（标签筛选按钮、分页按钮）

---

### P1-5：Admin 后台改进

- 所有时间列从 ISO 格式改为本地化（`dayjs` 或 `date-fns`，格式 `YYYY-MM-DD HH:mm`）
- 文章编辑器：封面图 URL 旁加 `[预览]` 按钮，点击弹出图片
- 资源编辑器：同上，图标和封面都加预览
- 文件管理器：文件夹树加层级缩进（`padding-left` 递增），区分父子目录
- 用户管理：增加用户名搜索框，昵称列从 `-` 改为 `display_name || username` 兜底

## 🟢 P2：细节打磨

- 主题切换按钮加 `aria-label="切换主题"`
- SSR 渲染时在 `<html>` 上注入初始主题 class（从 Cookie 读取），防 FOUC 闪烁
- 首页 Hero 的网格点阵图案从纯 CSS radial-gradient 改为 SVG pattern（更清晰）
- 页脚增加一段 ASCII art 小狐狸：

   ( o.o )
    > ^ <
  狐风轩汐の小屋 © 2026

## 执行要求

1. 先通读所有需要修改的文件，规划改动范围
2. 按 P0 → P1 → P2 顺序执行，每类修完做一次 `pnpm dev` 验证
3. 代码高亮方案选型：如果后端 `markdown` 库的 `codehilite` 不好用，换 `mistune`；如果前端已有的 MarkdownViewer 组件不支持，升级或用 `highlight.js` CDN
4. 改动完成后，写一份简短的 `docs/reports/glm52-polish-report.md`，列出每个修复项的改动文件和关键变更
5. 不要修改数据库结构、不要改 API 接口、不要改路由命名——只修改前端和后端的渲染层

### 灵活性留白

- 具体色彩数值可根据实际效果微调
- CSS 动画/过渡的快慢可以根据体验调整
- 如果某个方案实现难度过大，可以做简化版但不要完全不做
- 装饰元素的位置和数量可以自由发挥，核心是"有二次元感 + 有代码感"

# FwindEmiko-Web 视觉审计报告

> 审计日期：2026-06-24
> 审计对象：汐汐酱个人博客站（F.windEmiko.top）
> 技术栈：Nuxt3 + TailwindCSS + Element Plus + Vue3 Admin SPA
> 设计方向：二次元 + 代码极客风格融合，玻璃磨砂质感，深色基调，Live2D 角色

---

## 执行摘要

本次视觉审计覆盖 **公网前端 9 个页面**、**Admin SPA 7 个页面**，以及 **5 项专项审查维度**。发现多个页面存在**严重渲染故障**（内容空白），移动端体验和主题系统有改进空间，二次元与极客风格的视觉表达尚未充分释放。

| 类别 | 发现问题数 | 严重问题 |
|------|-----------|----------|
| 公网页面 | 5 | 博客详情、资源详情、登录、注册页面空白 |
| Admin SPA | 5 | 时间格式、编辑器渲染、目录树层级等问题 |
| 专项审查 | 8 | 无语法高亮、Live2D 遮挡、主题持久化等 |

## 第 4 步：专项审查

### A. 二次元风格融入度

**评级：Moderate（中等）**

| 检查项 | 状态 | 说明 |
|--------|------|------|
| Live2D 角色 | ✅ 存在 | 基于 PixiJS 的 Mao 模型，支持拖拽、点击气泡、视线追踪、口型同步，是核心二次元元素 |
| ACG 插画/Pixel Art | ❌ 缺失 | Hero Banner 无动漫插画或像素艺术 |
| 动漫配色 | ⚠️ 偏弱 | 主色为科技蓝 `#38bdf8`，仅有少量 `purple-500/10` 渐变点缀，非典型 pastel 马卡龙配色 |
| 装饰图案 | ❌ 缺失 | 无樱花、星星、对话框角标等 ACG 视觉纹理 |
| 动漫字体 | ❌ 缺失 | 使用 Noto Sans SC 标准黑体，未使用 ZCOOL KuaiLe 等动漫风格字体 |
| 站点命名 | ✅ 体现 | "狐风轩汐の小屋"使用日语假名，具有 ACG 文化指向 |

**结论：** Live2D 看板娘是唯一的强二次元标识，其余 UI 更偏向现代玻璃态科技风格。看板娘与整体科技感的融合度尚可，但缺乏更多二次元视觉元素的呼应。

**建议：**
1. Hero 区域增加 ACG 风格插画或 Pixel Art 背景
2. 引入 1-2 个动漫风格装饰字体用于标题
3. 增加 subtle 的二次元纹理（如星空、粒子、对话框装饰）

---

### B. 代码极客风格

**评级：Weak（较弱）**

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 语法高亮 | ❌ 缺失 | 未集成 Shiki、PrismJS 或 Highlight.js，代码块无语法高亮 |
| 代码块背景 | ⚠️ 基础 | 使用 `var(--bg-tertiary)`（深色 `#1f2937`，浅色 `#f1f5f9`），无专门主题 |
| 终端绿/黑客配色 | ❌ 缺失 | 无 Matrix 绿或终端绿强调色 |
| 等宽字体装饰 | ❌ 缺失 | 全局使用 Noto Sans SC，标题无等宽字体 |
| 终端风格组件 | ❌ 缺失 | 无命令行符号装饰、ASCII 艺术、二进制装饰 |
| 科技点阵背景 | ✅ 存在 | Hero 区域有 `radial-gradient` 32px 网格点阵图案 |

**结论：** 当前代码块仅为基础样式，缺乏技术博客应有的"代码氛围"。

**建议：**
1. 集成 Shiki 或 PrismJS 实现语法高亮（推荐 Dracula / One Dark / Monokai 主题）
2. 为代码块增加终端风格装饰（如 macOS 窗口圆点、行号、`$` 提示符）
3. 引入 JetBrains Mono / Fira Code 作为代码字体
4. 增加一个"终端模式"或"黑客模式"的彩蛋主题

### C. 玻璃磨砂效果 (Glassmorphism)

**评级：Good（良好）**

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 设计系统 | ✅ 完整 | `bg-glass`、`backdrop-blur-glass: 14px`、`border-glass-border`、`shadow-glass`、`rounded-glass: 14px` |
| 应用范围 | ✅ 广泛 | 导航栏、卡片、登录表单、筛选器、聊天气泡、Live2D 气泡均使用 |
| 深色模式 | ✅ 优秀 | `rgba(255,255,255,0.04)` 半透明低反光背景配合 14px 模糊，层次分明 |
| 浅色模式 | ⚠️ 可加强 | `rgba(255,255,255,0.65)` 在浅色背景下偏白、不够通透，缺少彩色折射效果 |
| 渐变叠加 | ⚠️ 可加强 | 缺少 `bg-gradient-to-br from-white/10 to-white/5` 等增强玻璃质感的渐变 |

**结论：** 玻璃磨砂体系完整且应用广泛，深色模式下表现优秀，浅色模式下通透感可进一步加强。

**建议：**
1. 浅色模式下增加微妙的渐变叠加和彩色边缘折射
2. 为玻璃卡片增加 hover 时的光泽扫过效果（glare effect）
